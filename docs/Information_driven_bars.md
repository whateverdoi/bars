# 信息驱动的Bars说明

## 2.3.2 信息驱动的Bars

信息驱动的Bars的目的是在市场上有新信息到达时进行更频繁的采样。在这一背景下，"信息"这个词是按微观结构意义使用的。市场微观结构理论赋予不平衡有符号交易量的持久性特别重要的地位，因为这种现象与知情交易者的存在相关联。通过将采样与知情交易者的到达同步，我们可能能够在价格达到新的均衡水平之前做出决策。

本文档探讨如何使用各种信息到达指标来对Bars进行采样。

---

## 2.3.2.1 Tick Imbalance Bars

### 基本概念

考虑一个tick序列 $\{(p_t, v_t)\}_{t=1,...,T}$，其中 $p_t$ 是与tick $t$ 相关联的价格，$v_t$ 是与tick $t$ 相关联的成交量。

**Tick Rule** 定义了一个序列 $\{b_t\}_{t=1,...,T}$，其中：

$$b_t = \begin{cases} 
b_{t-1} & \text{if } \Delta p_t = 0 \\
\frac{|\Delta p_t|}{\Delta p_t} & \text{if } \Delta p_t \neq 0
\end{cases}$$

其中 $b_t \in \{-1, 1\}$，边界条件 $b_0$ 设置为与紧邻前一个bar的终端值 $B_T$ 匹配。

### Tick Imbalance的定义

Tick Imbalance是指在时间 $T$ 处累积的有符号ticks（按tick rule签名）的总和：

$$\theta_T = \sum_{t=1}^{T} b_t$$

### 期望值计算

其次，我们计算bar开始时tick imbalance的期望值 $E_0[\theta_T]$：

$$E_0[\theta_T] = E_0[T](P[b_t = 1] - P[b_t=-1])$$

其中：
- $E_0[T]$ 是bar的期望长度（ticks数量）
- $P[b_t = 1]$ 是一个tick被分类为买单的无条件概率
- $P[b_t = -1]$ 是一个tick被分类为卖单的无条件概率

由于 $P[b_t = 1] + P[b_t = -1] = 1$，我们可以估计：
$$E_0[\theta_T] = E_0[T](2P[b_t = 1] - 1)$$

实际上，我们可以将 $E_0[T]$ 估计为来自先前bars的 $T$ 值的指数加权移动平均线，将 $(2P[b_t = 1] - 1)$ 估计为来自先前bars的 $b_t$ 值的指数加权移动平均线。

### Tick Imbalance Bar的定义

我们将 **Tick Imbalance Bar（TIB）** 定义为一个 $T^*$ 连续的tick子集，满足以下条件：

$$T^* = \arg \min_T \left\{ |\theta_T| \geq E_0[T]|2P[b_t = 1] - 1| \right\}$$

其中期望不平衡的大小由 $|2P[b_t = 1] - 1|$ 暗示。

### 工作原理

当 $\theta_T$ 的不平衡程度超过预期时，较小的 $T$ 会满足这些条件。因此，在知情交易者的存在下（导致单边交易），会更频繁地生成TIBs。

实际上，我们将TIBs理解为包含相等信息量的交易篮（无论成交量、价格或交易数量如何）。

## 2.3.2.2 基于成交量/成交额的不平衡Bars

### 概述

基于成交量的不平衡Bars（VIBs）和基于成交额的不平衡Bars（DIBs）的思想是扩展Tick不平衡Bars的概念。我们希望在成交量或成交额不平衡从我们的预期发散时进行采样。

基于相同的tick rule概念和边界条件 $b_0$（如我们为TIBs讨论的那样），我们定义一个步骤来确定下一个样本的索引 $T$。

### 时间T处的不平衡定义

首先，我们在时间 $T$ 处定义不平衡为：

$$\theta_T = \sum_{t=1}^{T} b_t v_t$$

其中 $v_t$ 可以代表以下两种情况之一：
- **VIB（成交量不平衡Bars）**：$v_t$ 表示交易的证券数量
- **DIB（成交额不平衡Bars）**：$v_t$ 表示交易的美元金额

您选择 $v_t$ 的含义取决于您是按成交量还是按成交额采样。

### 期望值计算

其次，我们在bar开始处计算 $\theta_T$ 的期望值 $E_0[\theta_T]$：

$$E_0[\theta_T] = E_0\left[\sum_{t=1}^{T} b_t v_t\right] - E_0\left[\sum_{t=1}^{T} b_t v_t\right]$$

$$= E_0[T](P[b_t=1]E_0[v_t|b_t=1] - P[b_t=-1]E_0[v_t|b_t=-1])$$

### 成交量分解

令 $v^+ = P[b_t=1]E_0[v_t|b_t=1]$ 和 $v^- = P[b_t=-1]E_0[v_t|b_t=-1]$，使得 $E_0[v_T] = E_0[\sum_t v_t] = E_0[v_t|]_{i=1}^T = E_0[v_t] = v^+ + v^-$。

您可以将 $v^+$ 和 $v^-$ 理解为将初始期望 $v_t$ 分解为买单贡献的部分和卖单贡献的部分。

因此：

$$E_0[\theta_T] = E_0[T](v^+ - v^-) = E_0[T](2v^+ - E_0[v_t])$$

### VIB/DIB的定义

我们将 **VIB或DIB** 定义为一个 $T^*$ 连续的tick子集，满足以下条件：

$$T^* = \arg \min_T \left\{ |\theta_T| \geq E_0[T]|2v^+ - E_0[v_t]| \right\}$$

其中期望不平衡的大小由 $|2v^+ - E_0[v_t]|$ 暗示。当 $\theta_T$ 的不平衡程度超过预期时，较小的 $T$ 会满足这些条件。

### 优势

这种基于成交量/成交额的不平衡Bar方法是信息驱动方法基于成交量和美元bars的类似方法，以及像其前辈一样，涉及以下问题：

1. **Tick Fragmentation（Tick碎片化）**：处理因市场分割导致的多个小ticks的问题
2. **Outliers（异常值）**：更有效地处理异常交易
3. **Corporate Actions（公司行为）**：与其前辈不同，该过程不依赖于恒定的bar大小。相反，bar大小是动态调整的

### 实际应用

在实践中，我们可以将 $E_0[T]$ 估计为来自先前bars的 $T$ 值的指数加权移动平均线，将 $(2v^+ - E_0[v_t])$ 估计为来自先前bars的 $b_t v_t$ 值的指数加权移动平均线。

## 2.3.2.3 Tick Runs Bars

### 概述

TIBs、VIBs和DIBs通过tick、成交量和成交额来监控order flow的不平衡。大型交易者会扫过order book、使用冰山订单或将母订单分成多个子订单，所有这些都会在 $\{b_t\}_{t=1,...,T}$ 序列中留下痕迹。

因此，监测整体成交量中的买单序列可能很有用，并在该序列与我们的预期偏离时进行采样。

### 当前运行长度的定义

首先，我们定义当前运行的长度为：

$$\theta_T = \max \left\{ \sum_{t|b_t=1}^{T} b_t - \sum_{t|b_t=-1}^{T} b_t \right\}$$

这衡量同一方向上连续ticks的最大长度。

### 期望值计算

其次，我们在bar开始处计算 $\theta_T$ 的期望值：

$$E_0[\theta_T] = E_0[T] \max\{P[b_t = 1], 1 - P[b_t = 1]\}$$

### 实际参数估计

在实践中，我们可以将 $E_0[T]$ 估计为来自先前bars的 $T$ 值的指数加权移动平均线，将 $P[b_t=1]$ 估计为来自先前bars的买单tick比例的指数加权移动平均线。

### Tick Runs Bar的定义

我们将 **Tick Runs Bar（TRB）** 定义为一个 $T^*$ 连续的tick子集，满足以下条件：

$$T^* = \arg \min_T \{\theta_T \geq E_0[T]\max\{P[b_t = 1], 1 - P[b_t = 1]\}\}$$

其中期望的运行长度由 $\max\{P[b_t=1], 1-P[b_t=1]\}$ 暗示。当 $\theta_T$ 比预期更不平衡时，较低的 $T$ 满足这些条件。

### 运行的概念

需要注意的是，在此定义中，运行允许序列中断。也就是说，我们测量每一方的tick计数，而不抵消它们（没有不平衡）。在bar形成的背景下，这被证明是比测量序列长度更有用的定义。

## 2.3.2.4 基于成交量/成交额的Runs Bars

### 概述

成交量Runs Bars（VRBs）和成交额Runs Bars（DRBs）扩展了上述Runs的定义到成交量和成交额交换。直观的想法是，我们希望在一方交易的成交量或成交额超过我们的预期bar时进行采样。

遵循我们对tick rule的惯例，我们需要确定bar中最后一次观测的索引 $T$。

### 成交量/成交额与运行的定义

首先，我们定义与运行相关的成交量或成交额为：

$$\theta_T = \max \left\{ \sum_{t|b_t=1}^{T} b_t v_t - \sum_{t|b_t=-1}^{T} b_t v_t \right\}$$

其中 $v_t$ 可以代表以下两种情况之一：
- **VRB（成交量Runs Bars）**：$v_t$ 表示交易的证券数量
- **DRB（成交额Runs Bars）**：$v_t$ 表示交易的美元金额

### 期望值计算

其次，我们在bar开始处计算 $\theta_T$ 的期望值：

$$E_0[\theta_T] = E_0[T]\max\{P[b_t=1]E_0[v_t|b_t=1], (1-P[b_t=1])E_0[v_t|b_t=-1]\}$$

### VRB/DRB的定义

我们将 **Volume Runs Bar（VRB）或Dollar Runs Bar（DRB）** 定义为一个 $T^*$ 连续的tick子集，满足以下条件：

$$T^* = \arg \min_T \{\theta_T \geq E_0[T]\max\{P[b_t=1]E_0[v_t|b_t=1], (1-P[b_t=1])E_0[v_t|b_t=-1]\}\}$$

其中期望的运行成交量由 $\max\{P[b_t=1]E_0[v_t|b_t=1], (1-P[b_t=1])E_0[v_t|b_t=-1]\}$ 暗示。当 $\theta_T$ 比预期更多运行或运行的成交量更大时，较低的 $T$ 会满足这些条件。

### 实际参数估计

在实践中，我们可以将 $E_0[T]$ 估计为来自先前bars的 $T$ 值的指数加权移动平均线，将 $P[b_t=1]$ 估计为来自先前bars的买单tick比例的指数加权移动平均线，$E_0[v_t|b_t=1]$ 估计为来自先前bars的买单成交量的指数加权移动平均线，$E_0[v_t|b_t=-1]$ 估计为来自先前bars的卖单成交量的指数加权移动平均线。

### 优势

与其前辈（TIBs、VIBs、DIBs）相比，Runs Bars有以下优势：

1. **处理单边交易**：在知情交易者存在时，能更有效地检测单边交易
2. **灵活的序列检测**：允许序列中断，更灵活地检测order flow运行
3. **处理Tick碎片化**：对于被多个小ticks分割的交易更加稳健
4. **处理异常值**：相比不平衡bars的方法，对异常交易的鲁棒性更强
5. **动态调整**：Bar大小会根据市场条件（单边交易活跃度）动态调整
6. **应对公司行为**：不依赖于恒定的bar大小，能更好地适应市场结构变化

## 实现要点

在Rust实现中，这些不同的Bar类型可以通过以下方式实现：

1. **BarType**：定义Bar的采样方式
   - 时间基础：按固定时间间隔采样
   - Tick基础：按固定tick数采样
   - **Tick Imbalance**：按tick不平衡阈值采样
   - **成交量基础**：按固定成交量采样
   - **成交额基础**：按固定成交额采样

2. **BarAggregator**：聚合器
   - 跟踪当前bar的状态
   - 计算是否达到采样条件
   - 生成完整的bar

3. **Tick**：基本单位
   - 时间戳
   - 价格
   - 成交量
   - 方向（通过tick rule确定）
